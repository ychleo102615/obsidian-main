# [560. Subarray Sum Equals K](https://leetcode.com/problems/subarray-sum-equals-k/description/) #card



我認爲這題很重要。很單純，技巧很優雅。我挺納悶 neetcode 上竟然沒有。

這題是在解週賽 [[3811. Number of Alternating XOR Partitions]] 時，Claude 建議練熟的基本題。

觀念：
```
	prefix :       ---------------------
	prev Prefix:   ------------
	target range:              ---------
```

prefix 代表的可以 sum 或是 XOR 結果。透過 `prefix - target = prevPrefix` ，找出：如果 prevPrefix 曾經出現的話，就可以利用該資料。

在這題中，存的是 subarray 的數量，「subarray sum 為 **特定數值**」時的數量。

所以例如：
目前 `nums[0...9]` 的 sum 是 100 ，我們的目標值是 70，那我們可以查詢是否有 `nums[0..k]` 的總和值為 `100 - 70 = 30`？
如果我們查到 `sum(nums[0..k]) === 30 ` 的答案有 3 個，代表目前有 3 個 subarray `sum(nums[k+1..9]) === 70`。

要記得如果先查詢，在將目前的資料存到 hash 裡面，否則會查到這回合的資料。
我寫錯過：
```
numx = [1]
k = 0
```
