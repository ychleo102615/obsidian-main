[1163. Last Substring in Lexicographical Order](https://leetcode.com/problems/last-substring-in-lexicographical-order/)

這題有一個很重要解題框架，沒想出來的話，用暴力解是會超時的。

這個框架是：題目的本質單純是在 `s` 中找到一個起點，從他開始的 substring 會是字典排序最大的。
因為 `s[i:]` 會大於 `s[i:-1]`。

所以題目變成`O(n)`的問題。

> [!NOTE]- 解法
> 解法是從頭開始比較，嘗試與目前 candidate 右鄰一個位置的起始座標比較。當分出勝負時，candidate 贏時把右鄰推進，右鄰贏時更新 candidate 為右鄰。

Claude:

每一步比較 `s[i+k]` 和 `s[j+k]`，有三種情況：

**情況 A：`s[i+k] == s[j+k]`（目前仍相同）**  
還分不出勝負，把比對長度延長一格：`k++`。

**情況 B：`s[i+k] > s[j+k]`（`i` 這邊比較大）**  
`j` 這個候選輸了。做 `j = j + k + 1`，`k = 0`。  
（可以直接跳過 `j` 到 `j+k` 這一段，後面「為什麼能整段跳過」會解釋。）

**情況 C：`s[i+k] < s[j+k]`（`j` 這邊比較大）**  
`i` 這個候選輸了，新的最佳候選變成 `j`。做 `i = max(i + k + 1, j)`，`j = i + 1`，`k = 0`。  
（為什麼是 `max` 而不是直接 `i = j`，稍後解釋。）

迴圈條件是 `j + k < n`。因為 `i < j`，所以 `i+k < j+k < n`，兩個索引都不會越界。結束後 `s[i..]` 就是答案。
### 為什麼情況 C 要用 `max(i+k+1, j)`

當 `s[i+k] < s[j+k]` 時，代表 `s[i..i+k]` 這一段（連同結尾那個較小的字元）整個輸給 `s[j..j+k]`。可以證明：**從 `i` 到 `i+k` 這些起始位置，全部都不可能是答案**，因為每一個位置 `i+t` 的後綴，都被它右邊對應的位置 `j+t` 的後綴壓過去。所以下一個要嘗試的起始位置至少是 `i+k+1`。

問題在於 `i` 和 `j` 這兩段有沒有重疊：

- **沒重疊**（`i+k < j`）：`i+k+1 ≤ j`，此時 `max` 取到 `j`，也就是把新冠軍設成剛剛贏的 `j`。合理。
- **有重疊**（`i+k ≥ j`）：被淘汰的區間 `[i, i+k]` 已經把 `j` 也包含進去了，代表 `j` 本身也被淘汰。這時不能設 `i = j`，必須跳到 `i+k+1`，此時 `max` 取到 `i+k+1`。

用 `max` 就同時處理了這兩種狀況。

### 為什麼情況 B 能一次跳過 `j` 到 `j+k`

當 `s[i+k] > s[j+k]`，`j` 輸了。而對於 `j+1, j+2, ..., j+k` 這些起始位置，它們的後綴開頭跟 `i+1, i+2, ..., i+k` 那些位置的後綴一模一樣（因為 `s[i..i+k-1] == s[j..j+k-1]`），但在後面某處就會分出高下、輸給那些以 `i` 開頭的後綴。而那些以 `i` 開頭的位置都在 `j` 之前、早就在候選範圍內了。所以 `j` 到 `j+k` 整段都不可能是答案，可以直接跳到 `j+k+1`。
