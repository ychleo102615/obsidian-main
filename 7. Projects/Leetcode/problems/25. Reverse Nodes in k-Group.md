我發現我卡在兩個地方：

### 一、迴圈條件
以往的寫法
```js
while (cur) {
    if (count < k) {
        count++;
        cur = cur.next;
        continue;
    }
}
```
我不太知道該怎麼解釋這段的行為。
我在結束條件上會搞混。
即使 linked list 是空的，這個算法仍然會數一次。
這感覺比較不對。

```js
while (cur.next) {
    count++;
    cur = cur.next;
    if (count !== k) {
        return;
    }
    // do the thing
}
```
上面的寫法，我比較好解釋：
當還有 elements 時（cur.next），把它加到一個虛擬的 queue 之中。
queue 滿了，就倒裝 queue 的順序。
