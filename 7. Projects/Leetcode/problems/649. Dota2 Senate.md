這題想了一陣子。我最初的的想法是使用兩個 array 和兩個指標，一個指標代表可以投票的 index ，更一個則是目前的進度 index。
但是我這個時候的思考模型是錯的，在 ban 對方的時候，不是從頭 ban ，而是 ban 對方下一個最快投票的 index 。所以可以投票的 index 這樣的做法是不可行的。

後來想到的方式是 double linked list 。有成功解出來，但是最終來說還是有點太過繞圈了？
我會把被 ban 的 index pop 出 linked list。發現其中一方在本輪投票已經沒得投之後，優勢方就可以從頭 ban 對方。
直到其中一邊的 linked list 為空。


但是我看到一個解答的方式非常優雅。使用資料結構 queue 就好了。
只是說有一個很重要的概念需要知道：他把循環投票的概念延伸成 append 到尾部的方式。

```js
queue.push(n++);
```
n 原本是資料長度，但在這邊延伸為：代表最新的投票順位。

所以有個 `RDD` 的例子，我一開始以為 R 在 queue 的演算法會失敗，因為我以為他的順位會一隻都是 `0`。用 append 最新順位的方式，第一個 R 的下一次投票順位是 `3` ，所以會輸給第二個 D（順位 `2`）。
