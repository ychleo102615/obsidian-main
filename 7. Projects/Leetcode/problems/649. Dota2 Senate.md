這題想了一陣子。我最初的的想法是使用兩個 array 和兩個指標，一個指標代表可以投票的 index ，更一個則是目前的進度 index。
但是我這個時候的思考模型是錯的，在 ban 對方的時候，不是從頭 ban ，而是 ban 對方下一個最快投票的 index 。所以可以投票的 index 這樣的做法是不可行的。

後來想到的方式是 double linked list 。有成功解出來，但是最終來說還是有點太過繞圈了？
我會把被 ban 的 index pop 出 linked list。發現其中一方在本輪投票已經沒得投之後，優勢方就等於可以 ban 對方
