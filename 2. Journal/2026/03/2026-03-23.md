#履歷回饋
1. ## 1

    但在看完目前的版本，會覺得假如從資深角度 (因為有六年多經驗了，這邊預設是要找資深的缺)，目前深度廣度上都有缺一些，不過要補上應該都不困難，只是在履歷上要展現出來。 深度的部分是要讓人知道你多懂某個技術點。例如個人專案有提到 CI/CD，但就只是一行帶過。如果要展現對前端 CI/CD，可以提到句體串了什麼，像是 enforces code quality through automated OXlint & OXfmt and Vitest unit testing，或是 Integrated bundle size monitoring and performance regression checks into the CI workflow, maintaining optimal Core Web Vitals 這種 (要串這些都不難，不過串完寫出來，就會讓人覺得你對網頁領域很熟) 廣度是指對網頁前端完整的掌握。舉例來說，目前有提到功能開發、單元測試，但是網頁前端的領域到很廣，例如用 Playwright 做 E2E 測試、確保應用有做好 a11y、有做好效能優化 (Core Web Vitals 各項跑分好看)。 如果在個人專案上把深與廣都補上，應該會讓審查履歷的人，覺得你確實懂網頁前端。
    
    ## 2
